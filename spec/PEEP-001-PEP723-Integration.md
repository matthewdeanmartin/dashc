Abstract
--------

This PEEP (Python Enhancement Enhancement Proposal) proposes integrating PEP 723 
"Inline script metadata" support into dashc, enabling the tool to extract and 
preserve dependency information and script metadata when converting Python scripts 
to bash-executable formats. This enhancement would make dashc-generated scripts 
self-documenting and enable better dependency management in distributed scripts.

Motivation
----------

PEP 723 introduces a standardized way to embed metadata directly in single-file 
Python scripts using a special comment block format. This metadata can include:

- Dependencies (pip requirements)
- Python version requirements  
- Tool configuration
- Script description and author information

Currently, dashc can convert Python scripts to bash-executable formats but loses 
this valuable metadata in the process. By preserving and utilizing PEP 723 metadata, 
dashc-generated scripts would become:

1. **Self-documenting**: Generated bash scripts would include the original dependencies and metadata
2. **Dependency-aware**: Could validate/install required packages before execution
3. **Version-safe**: Could check Python version compatibility
4. **Tool-compatible**: Support the emerging ecosystem of PEP 723-aware tools

PEP 723 Format Overview
-----------------------

PEP 723 defines inline metadata using this format:

```python
#!/usr/bin/env python3
# /// script
# requires-python = ">=3.11"
# dependencies = [
#   "requests<3",
#   "rich",
# ]
# ///

import requests
from rich import print

# ... rest of script
```

The metadata block:
- Starts with `# /// script` 
- Contains TOML-formatted metadata in comments
- Ends with `# ///`
- Is completely ignored by Python interpreter
- Can include `dependencies`, `requires-python`, and `[tool]` sections

from dataclasses import dataclass
from typing import List, Optional
import re
import tomllib  # Python 3.11+ or tomli backport

@dataclass
class ScriptMetadata:
    requires_python: Optional[str] = None
    dependencies: List[str] = None
    description: Optional[str] = None
    tool_config: dict = None
    raw_toml: str = ""

def extract_pep723_metadata(source_path: Path) -> Optional[ScriptMetadata]:
    """Extract PEP 723 metadata block from Python source file."""
    
    with open(source_path, 'r', encoding='utf-8') as f:
        content = f.read()
    
    # Find metadata block
    pattern = r'# /// script\n(.*?)\n# ///'
    match = re.search(pattern, content, re.DOTALL)
    
    if not match:
        return None
    
    # Extract and clean TOML content
    toml_lines = []
    for line in match.group(1).split('\n'):
        if line.startswith('# '):
            toml_lines.append(line[2:])  # Remove '# '
    
    toml_content = '\n'.join(toml_lines)
    
    # Parse TOML
    try:
        parsed = tomllib.loads(toml_content)
        return ScriptMetadata(
            requires_python=parsed.get('requires-python'),
            dependencies=parsed.get('dependencies', []),
            description=parsed.get('description'),
            tool_config=parsed.get('tool', {}),
            raw_toml=toml_content
        )

### 4. Integration Points

#### Core Module Changes

**Modify `dashc/single_file.py`:**

```python
def dashc(
    source_path: Path,
    plain_text: bool = False,
    shebang: str | None = None,
    preserve_metadata: bool = False,
    check_dependencies: bool = False,
    check_python_version: bool = False,
) -> str:
    """Generate bash script from Python file with optional PEP 723 support."""
    
    metadata = None
    if preserve_metadata or check_dependencies or check_python_version:
        metadata = extract_pep723_metadata(source_path)
    
    # Select appropriate template based on options
    if metadata and (preserve_metadata or check_dependencies or check_python_version):
        template_name = "script_with_metadata.j2"
    else:
        template_name = "script_plain.j2"  # existing template
    
    # Render with metadata context
    return render_template(
        template_name,
        source_file=source_path.name,
        metadata=metadata,
        check_dependencies=check_dependencies,
        check_python_version=check_python_version,
        python_command=generate_python_command(source_path, plain_text),
        shebang=shebang
    )
```

### 5. Example Usage Scenarios

#### Scenario 1: Simple Script with Dependencies

**Input `fetch_weather.py`:**
```python

# /// script
# requires-python = ">=3.8"
# dependencies = [
#   "requests>=2.25.0",
#   "click>=8.0.0",
# ]
# ///

import click
import requests

@click.command()
@click.option('--city', required=True, help='City name')
def weather(city):
    """Fetch weather for a city."""
    api_key = "demo_key"
    url = f"https://api.openweathermap.org/data/2.5/weather"
    response = requests.get(url, params={'q': city, 'appid': api_key})
    data = response.json()
    click.echo(f"Weather in {city}: {data['weather'][0]['description']}")

if __name__ == '__main__':
    weather()

# dependencies = [
#   "typer[all]>=0.9.0",
#   "httpx>=0.24.0",
#   "pydantic>=2.0.0",
# ]
# [tool.mypy]
# strict = true
# ///

import typer
from .core import process_data

app = typer.Typer()

@app.command()
def main(input_file: str = typer.Argument(...)):
    """Process input file with my_tool."""
    result = process_data(input_file)
    typer.echo(f"Processed: {result}")

if __name__ == "__main__":
    app()
```

**Command:**
```bash
dashc module my_tool --preserve-metadata --check-deps --check-python-version --out my_tool.sh
```

This would generate a script that:
1. Includes the PEP 723 metadata as comments
2. Validates Python version >= 3.9
3. Checks for typer, httpx, and pydantic before execution
4. Runs the compressed module with proper argument forwarding

- **Script Generation**: Scripts without PEP 723 metadata generate identical output
- **Template System**: New templates are additive, existing templates unchanged
- **Dependencies**: New dependencies are optional unless PEP 723 features are used

Security Considerations
-----------------------

### Dependency Installation Risks

**Risk**: Auto-installing dependencies from PEP 723 metadata could install malicious packages.

**Mitigation**:
- Make `--auto-install-deps` a very explicit, dangerous flag with warnings
- Default to `--check-deps` (validation only) rather than installation
- Add `--dry-run-deps` to show what would be installed without installing

### Metadata Injection

**Risk**: Malicious scripts could include fake or misleading metadata.

**Mitigation**:
- Validate TOML syntax strictly
- Warn users when auto-installation is enabled
- Include source file path in generated script comments for traceability

### Version Specification Attacks

**Risk**: Complex version specifiers could cause parsing errors or unexpected behavior.

**Mitigation**:
- Use the standard `packaging` library for version parsing
- Validate version specifiers before template rendering
- Graceful error handling with clear error messages
